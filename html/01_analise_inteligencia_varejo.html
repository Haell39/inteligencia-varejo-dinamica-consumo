1. Inteligência de Varejo: Impacto da Inflação e Eficiência Logística no PDV¶

🎯 Objetivo da Análise¶

Este projeto visa diagnosticar como variáveis macroeconômicas e falhas operacionais impactam o Sell-out (venda final ao consumidor). Investigaremos a correlação entre o aumento da inflação (IPCA) e a demanda de consumo, além de mensurar o "Custo Nordeste" através de indicadores de Ruptura Logística.

📊 Metodologia e Dados¶

Para esta análise, consolidamos duas fontes de dados robustas:

  • Olist (E-commerce Brasileiro): Dataset com 100.000 registros de vendas reais (2016-2018).
  • IBGE/SIDRA (API): Coleta em tempo real do IPCA mensal para os grupos de Consumo e Alimentos.

💡 Termos de Negócio Aplicados¶

  • Ruptura: Falha na disponibilidade do produto, aqui simulada pelo atraso na entrega (delivery delay).
  • Lead Time: Tempo total de ciclo logístico entre o pedido e o recebimento.
  • Sell-out: Performance real de saída dos produtos para o cliente final.

# 2.1 Coleta de Dados IBGE (Série Histórica IPCA)

print("🛰️ Iniciando requisição à API do SIDRA/IBGE...")

# Definição do período de análise (Jan/2016 a Dez/2018)
periodos = "|".join([f"{y}{m:02d}" for y in range(2016, 2019) for m in range(1, 13)])

# Endpoint para a Série Histórica do IPCA (Variação mensal)
url_ibge = f"https://servicodados.ibge.gov.br/api/v3/agregados/1737/periodos/{periodos}/variaveis/63?localidades=N1[all]"

try:
    response = requests.get(url_ibge)
    data = response.json()

    # Extração da série temporal da estrutura JSON
    raw_series = data[0]['resultados'][0]['series'][0]['serie']
    
    # Construção inicial do DataFrame
    ipca = pd.DataFrame(list(raw_series.items()), columns=['Mês (Código)', 'Valor'])

    # Feature Engineering: Padronização de datas e métricas
    ipca['Mês (Código)'] = pd.to_numeric(ipca['Mês (Código)'])
    ipca['IPCA_Mensal'] = pd.to_numeric(ipca['Valor'])

    # Formatação de chave temporal para o Join (YYYY-MM)
    ipca['AnoMes'] = (ipca['Mês (Código)'].astype(str).str[:4] + '-' + 
                      ipca['Mês (Código)'].astype(str).str[4:])

    # Seleção final de colunas estratégicas
    ipca = ipca[['AnoMes', 'IPCA_Mensal']]

    print("✅ Ingestão concluída. Dados formatados para análise.")
    display(ipca.head())

except Exception as e:
    print(f"❌ Falha na integração com IBGE: {e}")

2.2 Consolidação da Base Transacional (Olist)¶

Nesta etapa, realizamos o cruzamento de múltiplos datasets para criar uma visão única do cliente e do pedido. O objetivo é transformar dados brutos em indicadores de performance (KPIs) que permitam a análise regional e de satisfação.

🛠️ Transformações e Engenharia de Atributos:¶

  • Limpeza e Filtro: Foco exclusivo em pedidos com status delivered (entregues) para garantir a integridade do cálculo de lead time.
  • Cálculo de Ruptura (Atraso): Criação da variável delivery_delay_days para mensurar falhas na promessa de entrega.
  • Lead Time Físico: Cálculo do tempo total entre a compra e o recebimento pelo consumidor.
  • Enriquecimento (Merge): Unificação de dados geográficos (Estados), notas de satisfação (Reviews) e valores financeiros (Preço e Frete).

# CARREGAMENTO E CONSOLIDAÇÃO DOS DATASETS OLIST

print("📂 Carregando camadas de dados brutos...")
orders = pd.read_csv('../data/raw/olist_orders_dataset.csv')
order_items = pd.read_csv('../data/raw/olist_order_items_dataset.csv')
reviews = pd.read_csv('../data/raw/olist_order_reviews_dataset.csv')
customers = pd.read_csv('../data/raw/olist_customers_dataset.csv')

print("🛠️ Iniciando tratamento e feature engineering...")

# 1. Filtro de Integridade: Apenas pedidos entregues
orders = orders[orders['order_status'] == 'delivered'].copy()

# 2. Padronização de Séries Temporais
time_cols = ['order_purchase_timestamp', 'order_delivered_customer_date', 'order_estimated_delivery_date']
for col in time_cols:
    orders[col] = pd.to_datetime(orders[col])

# 3. Criação de KPIs de Performance Logística
# Diferença entre entrega real e estimada (Ruptura)
orders['delivery_delay_days'] = (orders['order_delivered_customer_date'] - orders['order_estimated_delivery_date']).dt.days

# Tempo total de ciclo (Lead Time)
orders['lead_time_days'] = (orders['order_delivered_customer_date'] - orders['order_purchase_timestamp']).dt.days

# 4. Criação de Chave para Cruzamento Macro (Ano-Mês)
orders['Mes_Codigo'] = orders['order_purchase_timestamp'].dt.year * 100 + orders['order_purchase_timestamp'].dt.month

# 5. Consolidação de Dimensões (Merges)
# Trazendo localização do cliente
orders = orders.merge(customers[['customer_id', 'customer_state']], on='customer_id', how='inner')

# Trazendo satisfação do cliente (último review por pedido)
reviews_clean = reviews.sort_values('review_creation_date').drop_duplicates('order_id', keep='last')
orders = orders.merge(reviews_clean[['order_id', 'review_score']], on='order_id', how='left')

# Trazendo volumetria financeira (Preço e Frete)
order_totals = order_items.groupby('order_id').agg({'price':'sum', 'freight_value':'sum'}).reset_index()
orders = orders.merge(order_totals, on='order_id', how='inner')

print("✅ Base Analítica consolidada com sucesso!")
# Exibindo colunas chave para a análise de Ruptura e Custo Regional
display(orders[['order_id', 'customer_state', 'delivery_delay_days', 'lead_time_days', 'review_score', 'freight_value']].head(3))

4. Conclusões e Descobertas Estratégicas (Revisadas)¶

1. Disparidade Logística Regional¶

Há uma diferença relevante entre regiões: Pernambuco apresenta lead time (~17,9 dias) e custo de frete significativamente maiores que São Paulo (~8,3 dias).
Implicação: existe um custo logístico regional que impacta eficiência e competitividade.

2. Impacto da Logística na Satisfação¶

Pedidos atrasados têm nota média de 2,27, enquanto entregas no prazo atingem 4,29.
Implicação: a logística é um fator crítico para a experiência do cliente e o desempenho de vendas.

3. Inflação vs Volume de Vendas¶

Não há evidência consistente de relação direta entre variações do IPCA e o volume de vendas no curto prazo.
Oscilações nas vendas parecem mais associadas a fatores como crescimento do e-commerce e sazonalidade (ex: Black Friday).

4. Elasticidade da Demanda¶

Os dados sugerem baixa sensibilidade das vendas ao IPCA no curto prazo.
Essa análise é exploratória e não controla variáveis como renda, promoções ou sazonalidade.

5. Direcionamento Estratégico¶

O impacto operacional (logística) é mais claro e imediato do que o impacto macroeconômico (inflação).
Prioridade: otimização logística tende a gerar maior retorno no curto prazo.
